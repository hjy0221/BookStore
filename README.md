# ITBookStore

ItBook API를 사용하여 도서 목록과 상세 정보를 보여주는 앱입니다.
요구사항에 맞춰 서드파티 라이브러리 없이 **UIKit**과 **Apple 프레임워크**만으로 구현했습니다.

## 1. 아키텍처: MVVM
**Model - View - ViewModel** 패턴을 적용했습니다.

* **선택 이유:** 처음에는 `ViewController`에 모든 코드를 작성했으나, 비즈니스 로직과 UI 코드가 섞여 유지보수가 어렵다고 판단했습니다. 역할 분리를 통해 가독성을 높이고, 추후 **단위 테스트(Unit Test)** 작성을 용이하게 하기 위해 MVVM을 도입했습니다.
* **구조:**
    * `View`: 화면을 그리고 사용자의 입력을 받습니다.
    * `ViewModel`: View에 보여질 데이터를 가공하고, `NetworkManager`와 통신합니다.
    * `Model`: 서버에서 받아온 데이터 구조(Codable)입니다.

## 2. 주요 기능 구현

### A. 이미지 캐싱 (Memory & Disk Cache) [요구사항 1]
외부 라이브러리 없이 효율적인 이미지 로딩을 위해 **이중 캐시 구조**를 직접 구현했습니다.
* **Memory Cache (`NSCache`):** 앱 사용 중 동일한 이미지를 다시 불러올 때 즉시 보여주기 위해 사용했습니다.
* **Disk Cache (`FileManager`):** 앱을 종료했다가 다시 켜도 이미지를 다운로드하지 않도록 파일 시스템에 저장했습니다. URL을 인코딩하여 파일명으로 활용했습니다.

### B. 오프라인 모드 지원 [요구사항 2]
네트워크가 끊긴 상황에서도 이전에 본 도서 정보를 볼 수 있습니다.
* **구현:** API 호출 성공 시 JSON 응답을 로컬 파일로 저장하고, 네트워크 실패 시 이 파일을 읽어오도록 처리했습니다.
* **의사결정:** 데이터 구조가 단순하기 때문에, 설정이 복잡한 `CoreData`보다는 가볍고 직관적인 **파일 시스템(FileManager)**을 사용하는 것이 효율적이라 판단했습니다.
